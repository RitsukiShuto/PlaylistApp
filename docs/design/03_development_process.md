# 開発プロセスとマルチエージェント構成 (Development Process)

本プロジェクトは、以下の専門性を持つAIエージェントが連携して開発を行う。

## 1. 実行・設計ロール
- **PM (プロジェクトマネージャー)**: 全体の進捗、優先順位、仕様の整合性を管理。
- **テックリード**: アーキテクチャ設計の監督、技術選定、コードレビュー方針の決定。
- **Spotify・APIエンジニア**: 外部API連携、認証フロー、データ同期ロジックの担当。
- **データ・グラフエンジニア**: 数値解析ロジック（マッピング計算）およびグラフDB設計の担当。
- **UI/UX・フロントエンドエンジニア**: Reactコンポーネント開発、インタラクション設計、スタイリングの担当。

## 2. 品質・運用ロール
- **構成管理**: ブランチ戦略、パッケージ依存関係、環境変数の整合性維持。
- **品質管理 (QA)**: テスト計画、自動テスト実装、バグ報告の検証。
- **ドキュメント管理**: 設計書とコードの乖離の監視、READMEやAPIリファレンスの整備。
- **SRE**: パフォーマンス監視、デプロイ自動化、Spotify APIのレート制限対応。

## 3. ワークフロー
1. **Brainstorming**: 各エージェントが専門的見地から仕様をレビュー。
2. **Design**: 詳細なタスク分割とインターフェース定義。
3. **Implementation**: TDD（テスト駆動開発）に基づいた段階的実装。
4. **Validation**: QAエージェントによる検証と、SREによる性能チェック。
